## Naming conventions

Styling hooks follow a specific naming convention to promote predictability and
consistency. All of our CSS custom properties adhere to the following convention:

```bash dark
COMPONENT_ELEMENT_CATEGORY_PROPERTY_ATTRIBUTE_STATE
```

- **Component** — [required] Name of the component
- **Element** — [optional] Name of the element of the component
- **Category** — [required] Indicates the category of the custom property.
- **Property** — [optional] Semantic UI property being described.
- **Attribute** — [optional] Semantic characteristic of a property.
- **State** — [optional] If necessary, the state of a custom property when used
in the context of interaction design. E.g. hover, focus, selected, active

Examples

```css dark
/* Component + Category + Property + Attribute */
button-color-background-primary

/* Component + Element + Category + Property */
accordion-header-font-size

/* Component + Category + Property + Attribute */
button-font-size-small

/* Component + Category + Property + Attribute + State */
button-color-background-primary-hover
```
