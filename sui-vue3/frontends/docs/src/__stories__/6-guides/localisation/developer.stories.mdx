## Vue-i18n

In SUI we use a plugin called `vue-i18n` which is peer dependency of `sui-core`. This swaps certain textual data depending on what locale is loaded in the consuming front-end application.
We generate a `.json` file for the English version, and then get the Translations team to provide translations for the other seven (7) languages we support,
in separate `.json` files and this plugin manages the loading and swapping of strings.

<div className="sui-storybook-highlight">
  For a full list of what is localised in each SUI component, <a href="https://siteminder.atlassian.net/wiki/spaces/SUI/pages/2309784582/SUI+localisation+keys">please visit our confluence</a> which lists our all of our strings.
</div>

Two components that don't use `vue-i18n` are `sm-date-picker` and `sm-calendar` - they both have their own implementation of the locale (e.g month names)
that is based on the browser's location using a Web API that is implemented in the browser.
