## Colour anatomy

The colour palette of Siteminder is used to create SUI's default themes. The neutral blue gray family is the default, with subtle shifts in value used to organize content into distinct zones.

All Siteminder products and experiences use the primary blue family as the primary action colour. Additional colours are used sparingly and on purpose.

## Extended palette

### Primary

A primary colour is the colour that appears frequently across screens and components and is used to emphasize actions. It is used for prominent action items such as buttons, text links, and other critical functional use cases.

Our primary colour related tokens are:
- **$primary-blue-highlight:** This is the hover state for the primary colour in the app-header only.
- **$primary-blue:** The primary colour is used to indicate interactions and primary actions on a page.
- **$primary-blue-dark:** This is the hover state for the primary colour.
- **$primary-blue-midnight:** This is the pressed state for the primary colour.

### Neutrals

The grey colour palette is used to create user interfaces, structure layouts, and convey hierarchy. Having a variety of options allows for complex layering of backgrounds, panels, cards, and so on.

Our neutral colour related tokens are:
- **$true-white:** This colour is used as a background colour in components (like cards) and also as the font colour in buttons with a darker fill (like the primary button).$grey-neu-black: SUI's version of 'black', this is used primarily as the colour of our font; also used in all focus rings to indicate what the user is focusing on when they tab target. For accessibility purposes only.
- **$grey-neu-light:** A legacy colour that is used in some components. Please avoid using this.$app-extra-4: This is the hover colour for table rows. It has been chosen to fall in with the app colours however it also needs to work on both a white background as well as app-light.
- **$blue-neu-light:** Used as an accent across our app, but primarily as the background colour in the wizard content area and vertical navigation.$blue-neu-med: This colour is used as a stroke to help visually define certain elements, such as cards, the tertiary button, dividers, etc.
- **$blue-neu-mid:** Used as the hover state in many of our components with a white background, such as the text button, tertiary button, many icon buttons, etc.
- **$blue-neu-dark:** Used primary to help accentuate the UI and also an alternative text colour to signify a count, i.e. Room Rates (5)

#### Disabled

Colour related tokens are:
- **$grey-neu-white:** This colour is used to indicate a disabled state on components and is used as a background colour in components that have a disabled state. The lightness of this colour is meant to meet contrast criteria when used with disabled-dark.
- **$grey-neu-med:** This colour is used to indicate a disabled state on components and is used as a stroke colour in components that have a disabled state.
- **$grey-neu-mid:** This colour is used to indicate disabled states used in our components. Do not use this colour with anything needs to meet a 4:5.1 contrast criteria against a white background, as it will fail.
- **$grey-neu-dark:** This colour is used to indicate a disabled state on components and is used as the text or icon colour in components that have a disabled state. The darkness of this colour is meant to meet contrast criteria when used with disabled-light.

## Use of Colour

https://www.w3.org/WAI/WCAG21/Understanding/use-of-color.html

It is essential not to rely solely on colour to convey information, indicate an action, elicit a response, or distinguish a visual element.
