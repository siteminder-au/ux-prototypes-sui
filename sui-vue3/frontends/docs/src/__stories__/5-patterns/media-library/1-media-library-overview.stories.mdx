## Product context

The Media Library pattern is provides an interface for assigning media files or images. (i.e. assigning images for room types).

This documentation outlines the guidelines and components necessary to implement a consistent media library across products.


## Anatomy

### Building the Media library with SUI

To implement this pattern, use the sm-dialog component in conjunction with the sm-media, sm-content-slider, sm-mixed-input, sm-button and sm-tabs components.

## Components and usage

### sm-media
The grid layout displaying thumbnails or previews of media files.

#### Guidelines
- Ensure that each media item displays a thumbnail
- Show an empty state if image fails to load.

<img src={smMedia} alt="sm-media component" />

### sm-media-item
An individual item within the media grid.

#### Guidelines
- Thumbnail: Display a preview image
- Selected State: Highlights the item when selected

<img src={smMediaItem} alt="sm-media item component" width="650px" />


### sm-content-slider
Shows the selected images from the library

#### Guidelines
- Button icons (arrows) - Buttons should disappear when there are no longer any images to scroll through.

<img src={smContentSlider} alt="sm-content-slider component"  />

### Upload button: sm-button
A button to upload new media files to the library.

#### Guidelines
- Use a prominent, easily accessible button.
- Support drag-and-drop functionality.

<img src={uploadButton} alt="sm-button as upload button" width="300px" />

### Assign button: sm-button
A button to assign selected images

#### Guidelines
- Use a prominent, easily accessible button.

<img src={assignButton} alt="sm-button as assign button" width="120px" />


### Search: sm-mixed-input
Tools for searching and filtering media items within the library.

#### Guidelines
- Include a search bar for keyword searches.
- Ensure real-time filtering and search results.

<img src={searchInput} alt="sm-mixed-input as search input field" width="320px" />
