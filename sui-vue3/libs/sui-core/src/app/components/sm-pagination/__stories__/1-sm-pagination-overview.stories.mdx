## Usage

- Pagination is most commonly used with search results.
- Pagination helps split larger collections of similar information onto several pages, which can help with browsability and scannability. Articles relating to a category or tag, content archives, and history or activity are all examples of multi-page collections.

## Anatomy
  ### Standard
  **1: Previous page button** - Enables users to go back to the previous page.

  **2: Page Jumper** - Consists of an input field to help users go directly to a page, and a text content indicating the total number of pages available.

  **3: Next page button** - Enables users to go to the next page.

  **4: Results indicator** - The total number of items/results.

  **5: Item page display (optional)** - The dropdown list of user-selected items to show per page.
  <img src={anatomyBasic} className="img-padding" alt="Standard pagination anatomy"/>


  ### Expanded
  **1: Previous page button** - Enables users to go back to the previous page.

  **2: Current page** - Indicates the current page number where the user is at.

  **3: Truncation** - Ellipsis are used to indicate the presence of additional pages. When users are in the middle of a page sequence with many pages, both next and previous, double truncation happens.

  **4: Next page button** - Enables users to go to the next page

  **5: Page Jumper** - Consists of an input field to help users go directly to a page, and a text content indicating the total number of pages available.

  **6: Item page display** - The dropdown list of user-selected items to show per page.

  **7: Results indicator** - The total number of items/results.
  <img src={anatomyExpanded} className="img-padding" alt="Expanded pagination anatomy"/>


## Variants
  ### Standard
  Standard pagination enables users to go back and forth between previous pages using the previous and next page buttons as well as allow users to enter a specific page they want to access immediately. Page and results indicator updates in real-time.
  <img src={variantBasic} className="img-padding" alt="Standard pagination variant"/>

  ### Expanded
  Expanded pagination enables users to jump to any page by selecting a page number button. We use an ellipsis to show that more pages exist.
  <img src={variantExpanded} className="img-padding" alt="Expanded pagination variant"/>

## Appearance
  ### Label
  For context, further labeling can be applied. If results equals only 1 page, the page display option can be omitted.
  <img src={appearanceLabel} className="img-padding" alt="Label appearance on pagination"/>

  ### Page display

## Behaviour
### Truncation
When there are more than 5 to 10 pages, truncation is used. An ellipsis is used to indicate the presence of more pages. When users are in the middle of a page sequence with numerous pages, both next and previous, double truncation happens.
<img src={behaviorTruncation} className="img-padding" alt="Truncation on pagination"/>

### Page display
The page display uses a select field to enable users to select the number of items to show on a page.
<img src={behaviorPageDisplay} className="img-padding" alt="Page display behavior on pagination"/>

### Previous and Next buttons
When the user is on/has selected the first page in the pagination range, the next page button advance the user to the next page in the sequence when selected. The previous page button is disabled in this use case and vice versa
<img src={behaviorButtons} className="img-padding" alt="Next and previous button behavior on pagination"/>
