## Usage
 <img src={overview} alt="Hint card" className="img-padding" />

### When to use
- You need to provide a subtle nudge or optimization tip.
- You want to guide users with actionable insights.
- You need a dismissible and non-intrusive way to surface information.

### Avoid when:
- The message is critical (use an alert instead).
- You need persistent information (consider a help card).

### Usage Guidelines
- Use one hint card per section to avoid clutter.
- Keep text concise and actionable.
- Only make the card dismissible if it's optional information.

 <img src={usage} alt="Hint card usage" className="img-padding" />

## Formatting

### Anatomy
 **1: Border** - 4px vertical colored bar indicating context.

 **2: Hint Icon** - A small lightbulb icon reinforcing the hint nature.

 **3: Content**
  - Title (Optional): Bold text summarizing the hint.
  - Body: Detailed message.

**4: CTA (Optional)** - A text link or button for further action.

**5: Close Button (Optional)** - Allows users to dismiss the hint.

<img src={anatomy} alt="Hint card anatmoy" className="img-padding" width={746} />

### Sizing
The height of the card will adjust depending on how many lines of text are within. The width of the card should always span the width of the page or container.

## Variants
Hint cards can be without heading, only body text which can be used for smaller hints.

### Default
Basic hint card with heading, body, and optional CTA.

## Behaviour

### Clicking action button
Redirects user or triggers an event.

### Closing a hint card
Card is dismissed and does not reappear, or product teams can set when the card will appear again.
