## Usage

### When to use
Breadcrumbs should be used when the user is faced with a large amount of content with two or more levels of content hierarchy. This component helps organise this hierarchy and allow the user to understand where they are within it.

Do not use breadcrumbs to replace the primary navigation of a page or product. It is purely a secondary navigation feature. It must only ever be used with two or more levels of content hierarchy.

Please do not use this as a form of multi-step progress indicator.


## Formatting

### Anatomy

**1: Page link -** This is the label of the page name. Labels are links, indicated by an underline.

**2: Separator -** Use a forward slash in text to separate the page labels. This can also be shown in icon form as a chevron/right arrow.

**3: Overflow (optional, not live) -** If width or screen space is a concern, then you can use the overflow feature. This truncates the levels if there are more than 3, or if you are on a touch device where space can be an issue.

**4: Current page -** The page the user is currently on. This is indicated by semi-bold/600 weight text.

<img src={anatomy} className="img-padding" alt="Anatomy of  breadcrumbs." />

### Placement
Breadcrumbs should always be at the top of the page, before the sm-page-title component. There should be a margin of 24px below it.
<img src={placement} className="img-padding" alt="Breadcrumbs placement." />

### Sizing
Breadcrumbs default sizing is `14pt`, or `para-sm`. Large sizing is `15pt/body`. The **large size ** should be used on any **touch device.**

## Variants

### Default
The default variant of sm-breadcrumbs. Everything is visible, with no hidden overflow.
<img src={variantDefault} className="img-padding" alt="Breadcrumbs default variant." width="280" />

### Overflow (not yet live)
Also a default variant of sm-breadcrumbs. Everything is visible, with no hidden overflow.

Use this variant when space is an issue or you have more than 4-5 levels of navigation hierarchy. Always use this on **mobile**.
