## Usage

  Use lists to break down information and make long blocks of text easier to read.

  Generally lists do not have columns, in the same way that a table has columns. If you require to present information and require a column layout, then it would be best to use **sm-table**.

## Anatomy

  **1: List** - Default state of the list item.

  **2: Header (optional)** - Name/title of the list.

  **3: Row** - Actual list-item.

  **4: List content** - Content of the list item.

  **5: Control (optional)** - Can display information and actions for list items such as radio buttons or checkboxes.

  **6: Visuals (optional)** - Displays supporting visuals to the content such as icon or image.

  **7: Actions (optional)** - Extra action displayed on the right of the list content.

  **8: Divider** - Border: 1px , color: $blue-neu-mid.

## Variants

  ### Standard list
  Essentially a row repeater that displays data as an ordered vertical collection of items. Rows in the list can be a single-line list or a multi-line list.
  <img src={typesStandard} alt="Standard list" className="img-padding-small" />
  <img src={typesStandardOld} alt="Standard list" className="img-padding" />

  ### Custom list
  The sm-list-item components default slot supports any amount of custom content. It can display different elements used in it, such as icons, text, and components such as Avatar, Badge, Checkbox, Icon, Progress, Switch, and so on.
  <img src={typesCustom} alt="Custom list" className="img-padding" />

  ### Draggable list
  A list can be reordered by dragging items. Use the draggable and list props to allow the user to sort the list.
  <img src={typesDrag} alt="Draggable list" className="img-padding-small" />
  <img src={typesDragOld} alt="Draggable list" className="img-padding" />

  ### Nested draggable list
  Items can be dragged into another item to create a nested list. Vice versa also applies - they can be dragged out of the nested list to create its own main list item.

  Use a combination of the group prop and a tree-like data structure to allow draggable lists to nest within each other.
  <img src={typesNested} alt="Nested draggable list" className="img-padding" width="501" />

  ### Nested draggable list across multiple containers
  Depending on the scenario there may be different logic applied to the positioning of list items.

  Use children: [] to set empty array to provide space for draggable children inside each group.
  <img src={typesMulti} alt="Nested draggable list" className="img-padding-small" width="450" />
  <img src={typesMultiOld} alt="Nested draggable list" className="img-padding" />
