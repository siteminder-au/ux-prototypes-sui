## Usage

  - Use the colour picker if the most common use case is to freely select any colour.
  - Using hexadecimal numbers, the colour picker can apply a custom colour to an element such as text or background colour. This is a common technique for theming.



## Anatomy

  **1: Colour slider** - A visual representation of which a user can choose colours from

  **2: Colour selector canvas** - A visual representation for adjusting the lightness and saturation of a selected colour

  **3: Colour handle** - A visual representation for adjusting the lightness and saturation of a selected colour

  **4: Active colour swatch** - A square container that shows the selected colour

  **5: Hex code input** - Displays the current colour settings, a hexadecimal value can also be inputted to select a colour

  **6: Button** - Primary button used to confirm colour selection

  **7: Container** - Wraps the colour picker elements

## Behaviours

  ### Display colour selection
  It is essential to clearly display the colour selection in real time when using the colour picker.
  <img src={behaviourDisplay} alt="Colour selection display" width="983" />

  ### States
  The hex code input should follow the states of the mixed input field component, the button should follow the button component states.
  <img src={behaviourStates} alt="Colour picker input states" width="983" />

  ### Mouse/touch
  Users use the mouse or touch to select a combination of saturation and lightness in the colour selector box (click and drag).

  Sliders are used to select the hue/colour values.
