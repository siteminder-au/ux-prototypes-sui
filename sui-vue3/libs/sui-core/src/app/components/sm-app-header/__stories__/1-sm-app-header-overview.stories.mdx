## Formatting

### Anatomy

**1: Dashboard link / App logo -** The app logo is also an icon button that will take you to the dashboard.

**2: Company name -** The 'SiteMinder' company name appears here in text. This can be customised by product or partner name.

**3: Smart Guide button -** This button will take you to our Smart Guide page.

**4: Property switcher -** Users may use this menu to switch properties, or go to property, billing or user management settings.

**5: Help button -** This button will take you to our help site, which opens in a new tab. However, this can also trigger a dropdown with help article links if required.

**6: Notifications button -** This button will trigger the notifications dropdown menu, which users can also use access the notifications page.

**7: Account menu button -** Access the user account menu via this button.

**8: Page links -** Our main navigational links sit along the bottom bar of the app-header.

### Nexus header: White-labelling
Currently, our app uses the nexus header. This is due to the fact that we still have users with partner accounts, and partner accounts have customisation settings which allow configuration of the colours and logo within the app-header.

The main difference between our SUI header and the nexus header is the button states. There are many restrictions when it comes to the amount of colour variables and how they are mapped via nexus. We are looking at moving away from nexus in the future.

For an in-depth look at the nexus header design, go [here](https://www.figma.com/file/cTcXdzx0DtNU2Pm8kkm7wL/?node-id=3282%3A82134).

## Variants

### Default
This is the default app-header that you will see throughout our app. This header works across all our breakpoints.

### Vue 2 app-header
The vue 2 app header is for products that have not yet fully migrated to SUI. We maintain a header that looks identical to our SUI one but is compatible with the old Sputnik design system. Products such as Insights still use this header. See it [here](https://sui.siteminder.systems/?path=/story/a-quick-start--using-storybook).

### Tablet compatible property switcher
So that the sub-menus within the 'Property Settings' and 'Billings' nav items in the property switcher are accessible via tablet, the property switcher has sub-menus which contain these sub-menu structures. Whereas on desktop the sub-menus are situated in the vertical-navigation.

**General behaviour**

- Clicking on the nav item with sub-menu will open a popover
- Clicking on the same nav item with the sub-menu opened will close the sub-menu
- Clicking on other items while the sub-menu is open will close the sub-menu
- Clicking inside the sub-menu will keep the sub-menu open
- Clicking outside the property switcher while a submenu is open will close both the property switcher and the sub-menu

### Button nav items
Button nav items are a type of nav item that can have child menu (sub-menu) structure, but the main parent item itself does not have an active state, or page link associated with it. Triggering this will open a sub-menu.

<img src={variantsButtonNavItems} className="img-padding" alt="App header button navigation items variant." width="720" />

### Link nav items
Link nav items are a type of nav item that can have child menu (sub-menu) structure, as well as an active state in the case that the parent menu has a link or page associated with it.
